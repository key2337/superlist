## 第四步：浏览器功能验证

### 4.1 新用户页面

打开浏览器访问 `http://39.105.88.230`

- [ ] 页面正常加载，样式正确（居中布局 + jumbotron）
- [ ] 页面标题 h1 显示 "Start a new To-Do list"
- [ ] 输入框可见，placeholder 为 "Enter a to-do item"

### 4.2 添加待办事项

在输入框中依次输入：

1. 输入 "买牛奶"，回车
2. 输入 "泡茶"，回车
3. 输入 "跑步"，回车

- [ ] 每条信息正确显示在下方 table 中
- [ ] 序号正确：1: 买牛奶、2: 泡茶、3: 跑步
- [ ] 页面 h1 显示 "Your To-Do list"
- [ ] 浏览器 URL 栏显示该用户的专属地址（如 `/lists/1/`）
- [ ] 页面样式保持正确

### 4.3 记下专属 URL 并关闭

- [ ] 复制当前 URL（如 `http://39.105.88.230/lists/1/`）
- [ ] 关闭浏览器页面

### 4.4 新用户验证

重新打开 `http://39.105.88.230`

- [ ] 页面显示空白待办页面 "Start a new To-Do list"
- [ ] 不显示之前的 "买牛奶"、"泡茶"、"跑步"
- [ ] 输入 "写代码"，回车
- [ ] 显示 "1: 写代码"
- [ ] URL 与之前不同（如 `/lists/2/`）

### 4.5 回访旧用户

打开之前记下的 URL（`/lists/1/`）

- [ ] 页面 h1 显示 "Your To-Do list"
- [ ] 之前的 "买牛奶"、"泡茶"、"跑步" 仍然显示
